# K线包含处理规则

## 一、包含关系的定义

**包含**：一根K线的高点和低点完全**被另一根K线的高点和低点包住**。

- 被包含：`K线n的高点 <= K线n-1的高点` 且 `K线n的低点 >= K线n-1的低点`
- 反向包含：`K线n的高点 >= K线n-1的高点` 且 `K线n的低点 <= K线n-1的低点`

## 二、包含处理规则

### 2.1 前提条件

始终基于**前一根非包含K线**的方向来处理。

### 2.2 向上处理

当前K线方向为**向上**（即前一根处理后的K线高点更高、低点也更高）：
- 遇到被包含的K线，取**两根K线中较高的高点和较高的低点**合并
- 公式：`new_high = max(K1.high, K2.high)`
- `new_low = max(K1.low, K2.low)`

### 2.3 向下处理

当前K线方向为**向下**（即前一根处理后的K线高点更低、低点也更低）：
- 遇到被包含的K线，取**两根K线中较低的高点和较低的低点**合并
- 公式：`new_high = min(K1.high, K2.high)`
- `new_low = min(K1.low, K2.low)`

## 三、算法流程

```
输入: K线列表 bars = [(open, high, low, close), ...]
输出: 包含处理后的K线列表 merged_bars

步骤:
1. 取第1根K线作为当前处理基准K线（current）
2. 从第2根K线开始，逐个处理后续K线：
   a. 判断current的方向（向上/向下）:
      - 向上: current.high > prev.high 且 current.low > prev.low
      - 向下: current.high < prev.high 且 current.low < prev.low
   b. 判断当前K线与current是否包含:
      - 检查是否互为包含
   c. 如果包含:
      - 根据方向取max或min进行合并
      - 合并后的K线作为新的current
      - 继续处理下一根K线
   d. 如果不包含:
      - 将current加入结果列表
      - 当前K线成为新的current
3. 将最后处理的current加入结果列表
```

## 四、边界情况处理

### 4.1 方向不明的情况
- 如果第一根K线无法确定方向，暂存为起始K线
- 等后续K线出现后确定方向再处理

### 4.2 连续包含
- 多根K线连续包含时，**逐对处理**
- A包含B → 合并成AB → AB包含C → 合并成ABC → ...
- 始终基于最新的合并结果判断方向

### 4.3 包含后再处理
- 如果合并后的K线与下一根K线仍未脱离包含关系
- 则按相同方向规则继续合并
- 直到出现非包含关系

## 五、验证用例

### 用例1：简单向上包含
```
K1: H=10, L=8  (向上)
K2: H=9, L=8.5 (被K1包含)
→ 合并结果: H=10, L=8.5
```

### 用例2：简单向下包含
```
K1: H=10, L=8  (向下)
K2: H=9, L=8.5 (反向包含K1)
→ 合并结果: H=9, L=8.5
```

### 用例3：连续包含
```
K1: H=10, L=8
K2: H=9.5, L=8.5 (包含于K1，向上处理→ H=10, L=8.5)
K3: H=9.8, L=8.8 (包含于H=10,L=8.5，向上处理→ H=10, L=8.8)
```
