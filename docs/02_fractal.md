# 分型识别规则

## 一、分型的定义

### 顶分型
三根连续K线，中间一根（不分方向）的**高点最高**，**低点也最高**。

```
     K2（最高点最高，最低点也最高）
    / \
   /   \
K1     K3（次高点，最低点最低）
```

条件：经过包含处理后
- `K2.high > K1.high` 且 `K2.high > K3.high`
- `K2.low > K1.low` 且 `K2.low > K3.low`

### 底分型
三根连续K线，中间一根的**低点最低**，**高点也最低**。

```
K1       K3（次低点，最高点最高）
 \       /
  \     /
   K2（最低点最低，最高点也最低）
```

条件：经过包含处理后
- `K2.low < K1.low` 且 `K2.low < K3.low`
- `K2.high < K1.high` 且 `K2.high < K3.high`

## 二、包含处理与分型的关系

**核心原则：必须在包含处理之后再判断分型。**

- 先对K线做包含处理（01_kline_merge.md的规则）
- 在**处理后的K线**上判断顶/底分型
- 包含处理后的K线才是"标准K线"

## 三、分型识别算法

```
输入: 包含处理后的K线列表 merged_bars
输出: 顶分型和底分型位置列表

步骤:
1. 从第2根K线到第倒数2根K线，依次检查每根K线
2. 对每根K线 K[i]：
   a. 检查是否是顶分型：
      判断 K[i-1], K[i], K[i+1] 是否构成顶分型
   b. 检查是否是底分型：
      判断 K[i-1], K[i], K[i+1] 是否构成底分型
3. 记录所有分型的位置（K线索引）
```

## 五、验证用例

### 用例1：标准顶分型
```
K1: H=9,  L=8.5  (正常)
K2: H=12, L=10   (高点最高，低点最高)
K3: H=10, L=9    (正常)
→ K2 是顶分型
```

### 用例2：标准底分型
```
K1: H=10, L=9    (正常)
K2: H=8,  L=7    (低点最低，高点最低)
K3: H=9.5, L=8.5 (正常)
→ K2 是底分型
```

### 用例3：包含处理后出现分型
```
原始K线:
K1: H=10, L=8
K2: H=9, L=8.5   (被K1包含)
K3: H=7, L=6

处理过程:
- K1和K2先做包含处理（向上方向）
- 合并后: H=10, L=8.5
- 再和K3比较: H=10, L=8.5 和 H=7, L=6

判断:
- K2(合并后)的高点10 > K1高点9... 等等
→ 按包含处理后的K线判断
```

### 用例4：非分型（中间K线不是极点）
```
K1: H=10, L=8
K2: H=11, L=8.5  (高点最高，但低点不是最高)
K3: H=9.5, L=8.8 (低点比K2高)
→ K2不是顶分型（K2.low < K3.low 不满足"低点最高"）
```

## 六、边界情况

1. **连续同向分型**：连续出现多个顶分型时，取最高那个
2. **分型间距**：顶分型和底分型之间至少间隔1根K线（不能相邻）
3. **方向必须交替**：顶分型后必须是底分型，底分型后必须是顶分型
4. **第一根和最后一根**：不判断分型（需要前后各1根K线）
