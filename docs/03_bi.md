# 笔的划分规则

## 一、笔的定义

**笔**：相邻的顶分型和底分型之间构成一笔。

- 先底后顶 → 向上一笔
- 先顶后底 → 向下一笔
- 所有的图形，都可以唯一地分解为**上下交替的笔的连接**

### 1.1 笔的几何结构

**向上一笔** = 底分型 + 上升K线 + 顶分型

```
      顶分型
      / \
     /   \
    /     \
   /       \
  /  上升K线 \
 /           \
底分型         \
```

**向下一笔** = 顶分型 + 下降K线 + 底分型

```
顶分型
 \
  \  下降K线
   \
    \
     \
      \
       \
       底分型
```

这里的"上升K线"或"下降K线"可以是**1根或多根**（经过包含处理后），只要一直保持定义就可以。

### 1.2 基本约束

| 约束 | 说明 |
|------|------|
| **一顶一底** | 必须是一个顶分型和一个底分型交替，不能出现顶顶或底底 |
| **至少1根K线** | 顶和底之间至少要有1根经过包含处理的K线 |
| **方向交替** | 顶分型后必须是底分型，底分型后必须是顶分型，交替出现 |
| **结合律** | 顶和底之间不能共用一根K线（违反结合律的不算一笔） |

## 二、笔划分的完整算法

### 2.1 预处理条件

笔的划分必须建立在**包含处理后的K线**和**分型识别完成**的基础上：

1. 先对原始K线做包含处理（`01_kline_merge.md`）
2. 在包含处理后的K线上识别顶底分型（`02_fractal.md`）
3. 在分型基础上划分笔

### 2.2 核心算法

```
输入: 分型列表 fractals（按时间升序排列）
      包含处理后K线列表 merged_bars
输出: 笔列表 bi_list

步骤:
1. 初始化：
   - 从第一个分型开始作为起始分型（start）
   - 笔列表初始为空

2. 遍历分型（从第2个分型开始到最后一个）：
   a. 当前分型 current = fractals[i]
   
   b. 检查 start 和 current 是否满足笔的条件：
      - 必须是不同类型（一顶一底）
      - 顶和底之间至少间隔1根K线
      - 不能共用K线（不违反结合律）
   
   c. 如果满足笔的条件：
      - 记录一笔 (start → current)
      - 将 current 设为新的 start
      - 继续遍历下一个分型
   
   d. 如果不满足（两个同向分型）：
      - 取两者中更高的顶 / 更低的底
      - 即"顶取最高，底取最低"
      - 较弱的那个分型被忽略
      - current 成为新的 start
      - 继续遍历

3. 返回笔列表 bi_list
```

### 2.3 分型取舍规则（核心难点）

#### 规则A：同向分型取极值

连续出现多个**顶分型**时，取**最高**的那个顶：
```
顶1 (H=10) → 顶2 (H=12) → 顶3 (H=11)
结果：取顶2 (H=12)
```

连续出现多个**底分型**时，取**最低**的那个底：
```
底1 (L=8) → 底2 (L=7) → 底3 (L=7.5)
结果：取底2 (L=7)
```

#### 规则B："先顶后底"和"先底后顶"

- **先顶后底** → 形成向下一笔
- **先底后顶** → 形成向上一笔
- 必须是交替的，不能顶顶相连或底底相连

#### 规则C：两个顶/底之间无其他分型

根据缠师65课：两个顶或底中间没有其他顶和底 → 第一个顶/底后的转折级别太小，不足以构成考察对象 → **第一个顶/底可以忽略不算**。

```
顶1 → 无其他分型 → 顶2
结果：顶1被忽略，从顶2开始
```

```
底1 → 无其他分型 → 底2
结果：底1被忽略，从底2开始
```

### 2.4 特殊处理

#### 新笔定义（放宽版本）

在实际应用中，有些软件/交易者采用"新笔"定义：
- 顶底之间不需要中间K线也可以成笔
- 前提是顶底分型必须明显（中间虽然没有经过包含处理的K线，但K线的高低点之差足够）

**本算法中默认采用标准笔定义**（顶底之间至少1根K线），可以通过参数控制是否启用新笔定义。

#### 跳空缺口的处理

- 如果缺口包含在一笔之内（顺着该笔方向），不影响笔的划分
- 如果缺口是**逆着走势方向**的（如530缺口），则必须当成新一段的开始（在线段层面处理）

## 三、分型与笔的归属关系

每个分型（除了被忽略的）都唯一地属于相邻的两笔，是两笔之间的连接点。

```
笔1               笔2
  \               /
   \             /
    分型（膝盖）
   /             \
  /               \
```

比喻：**膝盖就是分型，而大腿和小腿就是连接的两笔**。

## 五、验证用例

### 用例1：标准向上一笔
```
包含处理后K线:
K1: H=8,  L=7    (底分型)
K2: H=9,  L=7.5  (上升K线)
K3: H=11, L=9    (顶分型)

→ 向上一笔 (8→11)
```

### 用例2：标准向下一笔
```
包含处理后K线:
K1: H=11, L=9    (顶分型)
K2: H=10, L=8    (下降K线)
K3: H=8,  L=7    (底分型)

→ 向下一笔 (11→7)
```

### 用例3：连续顶分型取最高
```
分型序列:
底1 (L=8) → 顶1 (H=12) → 顶2 (H=13) → 底2 (L=9)

处理:
- 顶1和顶2连续出现，取较高的顶2 (H=13)
- 底1 → 顶2 → 一笔 (8→13)
- 顶2 → 底2 → 一笔 (13→9)
```

### 用例4：连续底分型取最低
```
分型序列:
顶1 (H=15) → 底1 (L=10) → 底2 (L=9) → 顶2 (H=14)

处理:
- 底1和底2连续出现，取较低的底2 (L=9)
- 顶1 → 底2 → 一笔 (15→9)
- 底2 → 顶2 → 一笔 (9→14)
```

### 用例5：顶底之间无中间K线（标准笔不成立）
```
K1: H=12, L=10 (顶分型)
K2: H=9,  L=8  (底分型)

标准笔: 不成立（中间无K线）
新笔:  根据参数可选
```

## 六、边界情况

### 6.1 首尾分型处理
- 第一根K线的前后没有完整K线构架 → 不参与笔划分
- 最后一根K线同理 → 不参与笔划分
- 笔的划分以完整分型为基础

### 6.2 笔方向的确定性
- 向上一笔必然以底分型开始、顶分型结束
- 向下一笔必然以顶分型开始、底分型结束
- 方向由起始和结束分型决定，不受K线实体影响

### 6.3 成笔的最小移动
- 标准笔：顶底之间至少1根经过包含处理的K线
- 这意味着笔至少由**3根**经过包含处理的K线构成
- 注意：这3根K线可能是原始K线经过包含合并后的结果

### 6.4 笔的长度
- 笔没有最大长度限制
- 只要始终保持方向不变（一直向上或一直向下）
- 可以有无穷根K线构成一笔（例如单边上涨行情）

### 6.5 分型所属的笔
- 除了被忽略的分型之外，每个分型都属于前后两笔
- 顶分型：属于前一向上一笔的结束 + 后一向下一笔的开始
- 底分型：属于前一向下一笔的结束 + 后一向上一笔的开始

## 七、常见问题

### Q1: 为什么要求顶底之间至少1根K线？

缠师原话：顶和底之间还有一根K线，这是"一笔的最基本图形"。如果顶和底直接相连（没有中间K线），则违反结合律，不算一笔。在实际分析中，都要求顶和底之间至少有一根K线。

### Q2: 为什么连续同向分型要取极值？

这是为了保证笔的"唯一分解"。如果不取极值（最高顶/最低底），就会出现同一个走势有多种笔的划分方式，违反了缠论的客观性。

### Q3: "新笔"和"标准笔"有什么区别？

| 特性 | 标准笔 | 新笔 |
|------|--------|------|
| 顶底间隔 | 至少1根K线 | 允许0根 |
| 适用场景 | 严格几何定义 | 实际交易的宽松版 |
| 稳定性 | 高 | 较低（易被噪声干扰） |

建议：首次实现使用**标准笔**，后续可通过参数启用新笔。
